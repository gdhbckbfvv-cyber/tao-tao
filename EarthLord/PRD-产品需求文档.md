## 1. 产品概述

### 1.1 产品愿景

EarthLord 是一款基于真实地理位置的末日生存游戏，玩家通过在现实世界中行走来圈地、探索、搜刮资源，在废土世界中重建文明。产品融合了"宝可梦GO"的LBS玩法、生存游戏的资源管理机制，以及深度的社交互动系统。

### 1.2 核心价值主张

- **Walk-to-Earn**：通过真实世界的行走获得游戏收益，鼓励健康生活方式
- **真实地理映射**：将真实世界的地标转化为游戏资源点，虚实结合
- **社交协作**：基于地理位置的交易、聊天、频道系统，连接附近玩家
- **深度策略**：资源管理、建造规划、生存体征管理等多维度游戏性

### 1.3 目标用户画像

**核心用户群体：**
- **年龄**：18-35岁
- **性别**：男性为主（60%），女性（40%）
- **特征**：
  - 喜欢户外活动、运动
  - 对生存类游戏感兴趣
  - 有一定游戏经验
  - 愿意尝试新型游戏玩法
  - 关注健康生活方式

**用户需求**：
- 需要游戏化的运动激励
- 寻求新颖的游戏体验
- 希望在游戏中社交互动
- 追求收集、成就感

### 1.4 市场机会

- **LBS游戏市场**：宝可梦GO累计收入超60亿美元，证明市场潜力巨大
- **健康游戏化**：Keep估值百亿，运动App市场持续增长
- **末日题材**：《辐射》《饥荒》等生存游戏长盛不衰
- **差异化优势**：市场上缺少将LBS + 生存 + 深度策略结合的产品

### 3.2 GPS圈地系统

**功能描述**：用户通过GPS轨迹在真实世界圈占领地

**功能需求**：
- ✅ 实时GPS定位（精度10米以内）
- ✅ 轨迹绘制（蓝色路径线）
- ✅ 自动闭合检测（起点终点距离<20米）
- ✅ 面积计算（平方米）
- ✅ 防作弊机制：
  - 速度检测（超过15km/h无效）
  - 直线距离检测
  - 时间间隔检测
- ✅ 领地重叠检测（PostGIS空间查询）
- ✅ 领地上限：单个玩家最多10块领地
- ✅ 面积限制：单块领地1000m² - 100,000m²

**技术实现**：
- 地图框架：Apple MapKit（免费、全球可用）
- 位置服务：CoreLocation
- 数据库：Supabase + PostGIS（地理空间扩展）
- 轨迹算法：Polygon闭合算法、面积计算

**验收标准**：
- GPS定位精度 < 10米
- 轨迹绘制延迟 < 1秒
- 面积计算误差 < 5%
- 防作弊检测准确率 > 95%

### 3.4 POI发现与探索系统

**功能描述**：玩家在真实世界行走，发现并搜刮POI（兴趣点）获取资源

**功能需求**：
- ✅ POI自动发现：
  - 进入POI 50米范围内触发
  - 弹窗提示"发现兴趣点"
  - 显示POI名称、类型、距离
- ✅ POI类型与资源映射：
  - 医院/药店 → 药品（急救包、抗生素）
  - 超市/餐厅 → 食物（罐头、饮用水）
  - 工厂/工地 → 金属材料
  - 公园/林地 → 木材
  - 银行/珠宝店 → 稀有材料
- ✅ 搜刮机制：
  - 每个POI每24小时可搜刮1次
  - 搜刮奖励基于探索距离和时长
  - 资源数量随机（区间范围）
- ✅ 探索统计：
  - 探索距离（公里）
  - 探索时长（分钟）
  - 搜刮次数
  - 获得物品明细

**技术实现**：
- POI数据源：Apple MapKit POI（免费）
- 地理围栏：CoreLocation Geofencing
- 冷却时间：本地+云端双重验证

**验收标准**：
- POI识别准确率 > 90%
- 距离计算误差 < 10%
- 冷却时间准确性 100%

### 3.6 建造系统

  created_at TIMESTAMP

territories (

  area FLOAT,
  allow_trade BOOLEAN,
  created_at TIMESTAMP

  type TEXT,
  level INT,

  position POINT,
  created_at TIMESTAMP
)

-- 物品表
items (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  item_type TEXT,
  quantity INT,
  created_at TIMESTAMP
)

-- 交易表
trades (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  offer_item TEXT,
  offer_quantity INT,
  request_item TEXT,
  request_quantity INT,
  status TEXT,
  created_at TIMESTAMP
)

-- 消息表
messages (
  id UUID PRIMARY KEY,
  channel_id UUID,
  user_id UUID REFERENCES users(id),

  created_at TIMESTAMP
)

## 7. 商业化策略

### 7.1 变现模式

1. **订阅制收入（主要）**
   - 开拓者月卡：$9.99/月
   - 领主通行证：$19.99/月
   - 目标付费率：5%
   - 月流水预期：$50,000+

2. **内购道具（次要）**
   - 资源包：$0.99 - $9.99
   - 特殊建筑：$4.99 - $14.99
   - 装备升级：$2.99 - $9.99

3. **广告收入（可选）**
   - 非付费用户观看广告获得奖励
   - 预计ARPU：$0.1/用户/天

### 7.2 付费点设计

**痛点付费**：
- 背包容量不够 → 购买扩容
- 探索范围太小 → 订阅月卡
- 资源不足 → 购买资源包
- 建造速度慢 → 购买加速道具

**炫耀付费**：
- 专属建筑皮肤
- 特殊称号
- 排行榜标识

**便利付费**：
- 自动收集资源
- 建筑快速升级
- 批量操作

### 7.3 首次付费引导

- 新手福利：首次购买5折
- 7日登录送月卡体验券（3天）
- 完成成就赠送优惠券
- 限时促销活动

## 9. 风险与挑战

### 9.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| GPS定位不准 | 用户体验差 | 多次采样、卡尔曼滤波优化 |
| 地图加载慢 | 用户流失 | 预加载、缓存机制 |
| 服务器宕机 | 服务中断 | 高可用架构、自动扩容 |
| 作弊行为 | 游戏公平性 | 多重防作弊检测 |

### 9.2 市场风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 竞品压力 | 用户分流 | 差异化功能、社区运营 |
| 用户留存低 | 商业化困难 | 优化核心循环、增加社交粘性 |
| 付费率低 | 收入不足 | 优化付费点设计、AB测试 |

### 9.3 合规风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 隐私政策 | 审核不通过 | 严格遵守GDPR、CCPA |
| 未成年保护 | 法律风险 | 年龄验证、防沉迷系统 |
| 地图数据 | 版权纠纷 | 使用官方API（Apple Maps） |
