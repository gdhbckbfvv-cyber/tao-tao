## 1. 游戏概念

### 1.1 核心理念

**"用双脚丈量废土，用希望重建文明"**

EarthLord是一款将真实世界行走与虚拟游戏体验深度结合的末日生存游戏。玩家化身为末日幸存者，通过在现实世界中行走来：
- **圈占领地**：用GPS轨迹圈地，建立自己的生存基地
- **探索废墟**：走到真实地标，搜刮物资
- **建设家园**：消耗资源建造建筑，发展生存基地
- **社交互动**：与附近玩家交易、聊天、组建联盟

### 1.2 独特卖点（USP）

1. **真实世界映射**：真实地标 = 游戏资源点，虚实融合
2. **Walk-to-Earn**：走路即游戏，健康即收益
3. **深度策略**：不是简单的计步器，而是复杂的生存策略游戏
4. **社交绑定**：基于地理位置的社交，连接现实中的邻居
5. **希望朋克美学**：不是绝望的末日，而是重建的希望

### 1.3 目标玩家

**核心玩家画像**：
- 喜欢户外活动、散步、骑行
- 对生存类游戏感兴趣（《饥荒》《方舟》《Rust》玩家）
- 有LBS游戏经验（玩过《宝可梦GO》《Ingress》）
- 追求收集、成就感
- 年龄18-35岁，男性为主

### 1.4 玩家动机分析（八角行为分析法）

| 动机 | 游戏设计对应 |
|------|-------------|
| 史诗意义与使命感 | 世界观：重建文明，成为新世界的开拓者 |
| 发展与成就 | 成就系统、等级系统、排行榜 |
| 创意授权与反馈 | 自定义领地名称、建筑摆放、策略选择 |
| 拥有感与占有欲 | 圈地占领、物品收集、建筑建造 |
| 社交影响与关联性 | 排行榜、交易系统、频道聊天、联盟 |
| 稀缺性与渴望 | 稀有资源、限时活动、排行榜奖励 |
| 未知性与好奇心 | POI探索、随机掉落、探索新地标 |
| 损失与逃避心 | 生存体征衰减、领地回收机制 |

## 2. 世界观与故事

### 2.1 背景故事

**时间线**：

```
2048年
│
├─ "终末战争"爆发
│  全球文明在72小时内崩塌
│  99%人口消失
│
↓
2051年（游戏开始）
│
├─ 幸存者从避难所走出
├─ 《新开拓者协议》颁布
│  "凡以双足丈量之无主土地，皆为开拓者所有"
│
↓
现在
玩家成为新世界的开拓者
重建家园，寻找希望
```

### 2.2 世界观设定

**世界状态**：
- 旧世界的秩序已崩塌，政府、货币、法律失效
- 大部分城市成为废墟，但基础设施仍可使用
- 自然开始重新占领城市，绿色植物在废墟中生长
- 幸存者分散各地，通过无线电联系

**新世界法则 - 《新开拓者协议》**：
1. **领地法则**："凡以双足丈量之无主土地，皆为开拓者所有"
2. **资源法则**：探索者有优先搜刮权
3. **交易法则**：以物易物，公平交换
4. **生存法则**：适者生存，互助共赢

### 2.3 美术风格 - 希望朋克（Hopepunk）

**核心理念**：不是绝望的末日，而是重建的希望

**视觉元素**：
- **废土元素**：
  - 破败的建筑、锈蚀的金属
  - 暗灰色调、沙尘天气效果
  - 裂纹、涂鸦、废墟纹理
- **希望元素**：
  - 新生的绿色植物破土而出
  - 温暖的光线（黄昏、日出）
  - 霓虹色标识（绿色、橙色）
  - 重建的痕迹（修复的建筑）

**配色方案**：
- 主色调：暗灰 (#2C2C2C)、军绿 (#4A5E3B)
- 强调色：霓虹绿 (#39FF14)、橙黄 (#FFA500)
- 辅助色：锈红 (#A0522D)、钢蓝 (#4682B4)

**参考作品**：
- 《辐射》系列（废土风格）
- 《最后生还者》（末日城市）
- 《赛博朋克2077》（霓虹色调）
- 《地平线：零之曙光》（自然重生）

### 3.2 每日游戏流程

**典型玩家一天的游戏流程**：

```
早上 08:00
├─ 登录游戏
├─ 查看每日任务
├─ 检查生存体征（饱食度、水分）
└─ 上班路上探索POI（搜刮早餐店、便利店）

中午 12:00
├─ 午休时建造/升级建筑
├─ 查看交易列表
└─ 公共频道聊天

下午 18:00
├─ 下班路上探索新POI
└─ 圈一块新领地（散步30分钟）

晚上 21:00
├─ 整理背包
├─ 发布交易挂单
├─ 查看排行榜
├─ 完成每日任务
└─ 领取奖励

睡前 23:00
├─ 使用食物/水恢复体征
├─ 查看明日任务
└─ 退出游戏
```

### 3.3 玩家成长路径

```
新手阶段（1-7天）
├─ 学会圈地（完成第一块领地）
├─ 学会探索（搜刮3个POI）
├─ 学会建造（建造第一个篝火）
├─ 学会交易（完成第一笔交易）
└─ 达成新手成就，获得初始资源

发展阶段（8-30天）
├─ 扩大领地（圈占2-5块领地）
├─ 建设基地（建造10+建筑）
├─ 积累资源（背包达到50+物品）
├─ 社交互动（加入3个频道）
└─ 考虑付费（月卡体验）

成熟阶段（31-90天）
├─ 优化布局（建筑升级至Tier 2-3）
├─ 资源富余（开始大量交易）
├─ 排行榜竞争（冲击Top 100）
├─ 建立联盟（创建私人频道）
└─ 稳定付费（订阅续费）

忠实玩家（90天+）
├─ 领地面积Top 10
├─ 建筑数量Top 10
├─ 探索距离Top 10
├─ 社区KOL（频道管理员）
└─ 长期付费用户
```

## 4. 游戏系统设计

### 4.1 圈地系统（Territory System）

**设计目标**：
- 让玩家有"拥有感"：这是我的土地
- 鼓励户外活动：必须真实行走才能圈地
- 防止作弊：多重检测机制

**核心机制**：

1. **圈地流程**
   ```
   步骤1: 点击"开始圈地"
   步骤2: 开始行走，GPS记录轨迹
   步骤3: 轨迹在地图上显示为蓝色线条
   步骤4: 走回起点附近（<20米），自动闭合
   步骤5: 系统计算面积，检测合法性
   步骤6: 合法则占领成功，非法则提示失败
   ```

2. **合法性检测**
   - ✅ 速度检测：平均速度 < 15km/h（防止开车作弊）
   - ✅ 轨迹检测：不能是直线往返（防止模拟器）
   - ✅ 面积限制：1,000m² - 100,000m²
   - ✅ 重叠检测：不能与其他玩家领地重叠 > 10%
   - ✅ 数量限制：每个玩家最多10块领地

3. **领地属性**
   - **名称**：玩家自定义（1-20字符）
   - **面积**：平方米，自动计算
   - **边界**：Polygon多边形坐标
   - **建筑槽位**：最多20个建筑
   - **状态**：活跃/休眠/即将回收
   - **权限**：允许/禁止交易

4. **领地回收机制**（防止占地不用）
   - 30天内无任何活动（无建筑、无探索）→ 进入休眠
   - 90天内无活动 → 被系统回收，重新变为无主土地

**数值设计**：

| 等级 | 解锁条件 | 领地上限 | 单块最大面积 |
|------|----------|----------|-------------|
| Lv 1 | 初始 | 1块 | 10,000m² |
| Lv 5 | 圈地5次 | 3块 | 30,000m² |
| Lv 10 | 圈地10次 | 5块 | 50,000m² |
| Lv 20 | 圈地20次 | 10块 | 100,000m² |

### 4.2 探索系统（Exploration System）

**设计目标**：
- 将真实世界地标转化为游戏内容
- 鼓励玩家走出去探索陌生地方
- 提供持续的资源来源

**核心机制**：

1. **POI发现**
   - 玩家进入POI 50米范围内 → 自动触发发现
   - 弹窗显示："发现 [POI名称]"
   - 显示POI类型、距离、可搜刮状态

2. **POI类型与资源映射**

| POI类型 | 真实地标示例 | 掉落资源 | 稀有度 |
|---------|-------------|----------|--------|
| 医院/诊所 | 人民医院、XX诊所 | 药品、急救包、绷带 | 稀有 |
| 超市/商店 | 沃尔玛、711 | 罐头食品、饮用水、布料 | 常见 |
| 餐厅/咖啡馆 | 星巴克、麦当劳 | 食物、水、餐具 | 常见 |
| 工厂/工地 | 建筑工地、工业区 | 金属、木材、工具 | 普通 |
| 公园/绿地 | 中央公园、森林公园 | 木材、草药、水源 | 普通 |
| 银行/金店 | 中国银行、珠宝店 | 贵金属、电子元件 | 稀有 |
| 加油站 | 中石油、中石化 | 汽油、机油、塑料 | 普通 |
| 学校/图书馆 | XX小学、市图书馆 | 纸张、书籍、工具 | 普通 |

3. **搜刮机制**
   ```
   触发条件: 进入POI 50米范围内
   搜刮按钮: "搜刮此地点"
   冷却时间: 每个POI 24小时可搜刮1次
   奖励计算:
     基础奖励 = POI类型基础值
     距离加成 = 探索距离 × 0.1（每1km增加10%奖励）
     时长加成 = 探索时长 × 0.05（每1分钟增加5%奖励）
     随机浮动 = ±20%
   最终奖励 = 基础奖励 × (1 + 距离加成 + 时长加成) × 随机浮动
   ```

4. **探索统计**
   - 总探索距离（公里）
   - 总探索时长（小时）
   - 发现POI数量
   - 搜刮成功次数
   - 探索排行榜

**防作弊机制**：
- 每个POI有24小时冷却（云端验证，防止本地修改时间）
- GPS轨迹验证（必须真实移动到POI附近，防止虚拟定位）
- 速度检测（防止开车刷POI）

|---------|------|----------|------|

```
[建造中] ──建造完成──> [待激活] ──激活──> [运行中]
                         ↑                    │
                         │                    │
                         └────停用────────────┘
                                              │
                                              │损坏
                                              ↓
                                         [损坏状态]
                                              │

                                              ↓
```

- 升级时保留原建筑功能，效果增强

### 4.4 交易系统（Trade System）

**设计目标**：
- 促进玩家社交互动
- 让资源流通（经济循环）
- 基于地理位置（鼓励线下见面）

**核心机制**：

1. **交易发布**
   ```
   我提供: [罐头食品] × 10
   我需要: [药品] × 5
   位置: 自动获取（当前领地）
   可见范围: 进入领地100米内的玩家
   有效期: 24小时（过期自动下架）
   ```

2. **交易发现**
   - 玩家A在自己领地发布交易
   - 玩家A开启"允许交易"
   - 玩家B进入玩家A领地100米范围
   - 玩家B自动看到玩家A的交易列表
   - 玩家B可选择接受交易

3. **交易执行**
   ```
   步骤1: 玩家B点击"接受交易"
   步骤2: 检查双方背包是否有足够物品
   步骤3: 原子性事务：
          - 玩家A背包：-我提供的物品, +我需要的物品
          - 玩家B背包：+我提供的物品, -我需要的物品
   步骤4: 交易完成通知双方
   步骤5: 记录交易历史
   ```

4. **交易限制**
   - 同时最多发布10个挂单
   - 每笔交易后5分钟冷却（防刷）
   - 必须在发布者领地100米范围内（防止远程交易）
   - 交易双方必须都在线

**交易数据统计**：
- 总交易次数
- 成交率（成交/发布）
- 最常交易物品
- 交易伙伴列表

### 4.5 生存体征系统（Vital Signs System）

**设计目标**：
- 创造紧迫感（必须持续游玩）
- 资源消耗出口（避免资源过剩）
- 生存游戏的核心机制

**体征指标**：

1. **核心生命值（Health）**
   - 范围：0-100
   - 初始值：100
   - 衰减条件：当饱食度或水分<20%时，每小时-1
   - 恢复方式：使用急救包、在庇护所休息

2. **饱食度（Hunger）**
   - 范围：0-100%
   - 初始值：100%
   - 衰减速度：每小时-5%
   - 恢复方式：使用食物（罐头+20%、新鲜食物+30%）

3. **水分（Thirst）**
   - 范围：0-100%
   - 初始值：100%
   - 衰减速度：每小时-10%（比饱食度衰减更快）
   - 恢复方式：使用饮用水（+30%）

**状态Buff/Debuff系统**：

| 条件 | 状态名称 | 效果 |
|------|---------|------|
| 饱食度>80% & 水分>80% | "充沛" | 探索奖励+20%, 建造速度+10% |
| 饱食度50-80% & 水分50-80% | "正常" | 无效果 |
| 饱食度<50% or 水分<50% | "虚弱" | 移动速度-10%, 探索奖励-10% |
| 饱食度<20% or 水分<20% | "饥渴" | 移动速度-20%, 开始掉血 |
| 生命值<30% | "濒死" | 所有属性-20%, 无法建造 |

**死亡与复活**：
- 生命值降至0 → 角色死亡
- 死亡惩罚：背包随机丢失50%物品
- 复活机制：
  - 自然复活：24小时后自动复活
  - 道具复活：使用"复活石"立即复活（稀有道具）
  - 付费复活：花费金币立即复活

**定时衰减机制**：
```
后台定时器: 每10分钟触发一次
计算逻辑:
  当前时间 - 上次更新时间 = 经过时间（小时）
  饱食度 -= 经过时间 × 5
  水分 -= 经过时间 × 10
  if (饱食度 < 20 or 水分 < 20):
    生命值 -= 经过时间 × 1
```

- 复活：100金币
- 购买资源包：50-100金币

**宝石获取**：
- 稀有成就：5-10宝石
- 付费购买：$0.99 = 10宝石

**宝石消耗**：
- 加速建造：5宝石/小时
- 购买稀有道具：10-50宝石

### 10.3 交易经济

**交易需求**：
- 资源不平衡：探索者资源多但建筑少，建造者相反
- 地理限制：不同地区POI类型不同，资源分布不均
- 紧急需求：急需某种资源完成建造

**交易定价**：
- 市场自由定价（玩家自己决定交换比例）
- 推荐比例：
  - 常见资源 : 常见资源 = 1:1（木材 ↔ 石头）
  - 普通资源 : 常见资源 = 1:2（金属 ↔ 木材）
  - 稀有资源 : 普通资源 = 1:3（药品 ↔ 金属）
  - 传说资源 : 稀有资源 = 1:5（贵金属 ↔ 药品）

## 11. 付费设计

### 11.1 付费点设计哲学

**核心原则**：
- ✅ Pay-to-Progress（付费加速），而非Pay-to-Win（付费获胜）
- ✅ 免费玩家可以体验所有核心玩法
- ✅ 付费玩家获得便利性和时间优势，但不破坏平衡
- ✅ 不强制付费，自然引导付费

### 11.2 订阅制设计

**开拓者月卡（$9.99/月）**：
- ✅ 无限探索次数（免费用户每日10次上限）
- ✅ 背包容量+50%（100格 → 150格）
- ✅ 探索范围+50%（50米 → 75米触发POI）
- ✅ 每日赠送资源包（木材×30, 石头×20, 食物×10）
- ✅ 月卡徽章（个人页展示）

**领主通行证（$19.99/月）**：
- ✅ 包含开拓者月卡所有特权
- ✅ 背包容量翻倍（100格 → 200格）
- ✅ 专属建筑皮肤（金色篝火、豪华庇护所）
- ✅ 专属称号"领主"
- ✅ VIP客服（优先解决问题）
- ✅ 建造时间-50%
- ✅ 体征衰减速度-30%

### 11.3 一次性购买

**资源包**：
- 初级物资包（$0.99）：木材×50, 石头×30, 食物×20
- 中级物资包（$2.99）：木材×150, 金属×80, 药品×30
- 高级物资包（$4.99）：全资源×200, 稀有材料×10
- 稀有资源包（$9.99）：电子元件×50, 贵金属×20, 复活石×1

**特殊道具**：
- 建造加速器（$0.99）：立即完成当前建造
- 背包扩容（$2.99）：永久+50格
- 复活石（$1.99）：死亡时立即复活，无惩罚

### 11.4 付费引导策略

**首次付费优惠**：
- 新用户首次购买任意商品5折
- 完成新手引导后赠送$0.99优惠券

**月卡体验**：
- 7日登录奖励：月卡体验券（3天）
- 让玩家体验付费的便利性，促进转化

**限时促销**：
- 每周五"补给日"：资源包7折
- 节日活动：特殊限定道具

**痛点付费**：
- 背包满了 → 弹窗"背包容量不足，是否购买扩容？"
- 建造时间长 → "等待2小时，或使用加速器立即完成"
- 探索次数用完 → "今日探索次数已用完，订阅月卡可无限探索"

### 11.5 付费数据目标

| 指标 | 目标值 |
|------|-------|
| 付费率 | 5% |
| ARPU（平均每用户收入） | $0.50/月 |
| ARPPU（平均每付费用户收入） | $10/月 |
| 订阅占比 | 70%（订阅收入占总收入比例） |
| 首次付费转化时间 | 7天内 |

### 13.3 核心指标定义

| 指标 | 定义 | 计算公式 |
|------|------|----------|
| DAU | Daily Active Users，日活跃用户 | 当天至少登录1次的用户数 |
| MAU | Monthly Active Users，月活跃用户 | 当月至少登录1次的用户数 |
| 次日留存率 | 新用户第二天回来的比例 | 次日登录用户数 / 新增用户数 |
| ARPU | Average Revenue Per User | 总收入 / 总用户数 |
| ARPPU | Average Revenue Per Paying User | 总收入 / 付费用户数 |
| LTV | Lifetime Value，用户生命周期价值 | 用户在整个生命周期贡献的总收入 |
| 付费率 | 付费用户占比 | 付费用户数 / 总用户数 |

## 14. 结语

EarthLord不仅仅是一款游戏，更是一个关于**希望与重建**的故事。

在这个虚拟的末日世界中，玩家通过在现实世界中行走，一步步圈下属于自己的土地，在废墟中寻找资源,建造家园，与其他开拓者交易、合作，共同重建文明。

这是一款将**真实世界与虚拟游戏深度融合**的作品，让玩家在游戏的同时，也在现实中锻炼身体、探索城市、连接邻居。

**希望朋克**的美学告诉我们：即使在最绝望的废土上，只要还有希望的种子，文明就能重新生长。

每一位玩家都是新世界的开拓者，每一步都在书写新的历史。

**让我们用双脚丈量这片废土，用希望重建文明。**
